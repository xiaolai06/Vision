## 垃圾分类智能识别系统 — 项目指南

### 一、项目概述

本系统基于 Vue 3 + FastAPI + SQLite + PyTorch 构建，支持垃圾分类的**单物体分类**（MobileNetV2）和**多物体检测**（YOLOv8）两种识别模式，并提供数据标注、训练数据导出、模型切换和实时流识别等完整功能。

项目采用前后端分离架构：前端负责交互与可视化，后端负责模型推理与数据管理。

### 三、接下来你需要做的事

#### 第一步：下载真实数据集

当前项目中的训练数据是基于 1 张原始图片增强生成的（约 700 张），可以跑通整个训练流程，但模型精度有限。要获得实用的模型，需要下载真实数据集。

**推荐数据源（按优先级排列）：**

| 数据源 | 类别 | 规模 | 特点 | 下载地址 |
|--------|------|------|------|----------|
| 天池/华为云 | 4大类40小类 | ~645MB | 完美匹配项目4分类，最推荐 | https://tianchi.aliyun.com/dataset/175980 |
| 百度飞桨 | 4大类40小类 | 较大 | 同样4分类，质量高 | https://aistudio.baidu.com/datasetdetail/239490 |
| HuggingFace | 10类 ~14000张 | ~2GB | 需映射到4分类，下载可能较慢 | `khoaliamle/Garbage_Classification_YOLO` |
| Kaggle TrashNet | 6类 ~2500张 | ~200MB | 经典数据集，需映射 | 需 Kaggle API Token |
| ModelScope | 4大类265小类 | ~13GB | 超大规模，完整但太大 | `tany0699/garbage265` |

**天池数据集的40小类到4大类映射关系：**

```
0-9 号小类   → other（其他垃圾）：一次性快餐盒、污损塑料、骨头贝壳、烟蒂...
10-19 号小类 → kitchen（厨余垃圾）：菜帮菜叶、茶叶渣、剩饭剩菜、果皮...
20-29 号小类 → recyclable（可回收物）：纸盒、塑料瓶、金属罐、玻璃瓶...
30-39 号小类 → hazardous（有害垃圾）：电池、灯泡、过期药品、油漆桶...
```

下载后执行以下命令导入数据：

```bash
cd backend

# 导入天池数据集（自动解压 + 4分类映射 + train/val/test 划分）
python -X utf8 data/download_datasets.py --source tianchi --tianchi-path 你的下载路径/garbage.zip

# 同时生成检测训练数据（全图 bounding box 基线）
python -X utf8 data/download_datasets.py --source tianchi --tianchi-path 你的下载路径/garbage.zip --gen-detect

# 查看数据分布
python -X utf8 data/download_datasets.py --source stats
```

如果你有自己的图片（按类别文件夹组织），也可以用：

```bash
python -X utf8 data/download_datasets.py --source custom --custom-path 你的图片目录/
```

#### 第二步：训练分类模型（MobileNetV2）

```bash
cd backend

# 基础训练（30 epoch，约 20-40 分钟）
python -X utf8 training/train_classification.py --epochs 30 --batch-size 32

# 如果显存不够，减小 batch-size
python -X utf8 training/train_classification.py --epochs 30 --batch-size 16

# 完整参数说明
python -X utf8 training/train_classification.py --help
```

训练完成后模型自动保存到 `model/best.pt`，重启后端即加载真实模型。

#### 第三步：训练检测模型（YOLOv8）

```bash
cd backend

# 基础训练（50 epoch，约 1-3 小时）
python -X utf8 training/train_detection.py --epochs 50 --batch-size 16

# 使用更大的模型（精度更高但更慢）
python -X utf8 training/train_detection.py --epochs 100 --batch-size 8 --model yolov8s

# 训练中断后继续
python -X utf8 training/train_detection.py --resume
```

训练完成后最佳权重自动复制到 `model/best_multi.pt`。

#### 第四步：标注更精确的检测数据

全图 bounding box 只是基线方案。要训练出真正好用的检测模型：

1. 启动项目，用检测模型识别一些图片
2. 进入「标注」页面，对检测结果的 bbox 进行人工修正
3. 标注完成后，在「记录」页面选择「YOLO检测」或「COCO检测」格式导出
4. 用导出的数据替换 `data/detection/` 目录中的内容，重新训练

#### 第五步：验证完整流程

启动前后端，依次测试：

```bash
# 启动后端
cd backend
python -X utf8 main.py

# 启动前端（另一个终端）
cd frontend
npm run dev
```

测试清单：上传图片识别（分类模式）、切换检测模型识别多物体、实时摄像头流识别、查看记录表格和图表、进入标注页修正 bbox、导出训练数据。

### 四、训练脚本详细说明

#### train_classification.py

**核心参数：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--epochs` | 30 | 训练轮数，建议 30-50 |
| `--batch-size` | 32 | 批大小，显存不足时减小到 16 或 8 |
| `--lr` | 0.001 | 初始学习率，配合 CosineAnnealing 自动衰减 |
| `--weight-decay` | 1e-4 | L2 正则化系数 |
| `--pretrained` | True | 使用 ImageNet 预训练权重（强烈建议开启） |
| `--device` | auto | 自动检测 CUDA，也可手动指定 cpu 或 cuda |

**训练策略：**

- 数据增强：RandomResizedCrop、水平/垂直翻转、颜色抖动（亮度/对比度/饱和度/色调）、随机旋转 15 度
- 学习率调度：CosineAnnealingLR，从初始学习率平滑衰减到 1e-6
- 模型结构：MobileNetV2 + Dropout(0.3) + Linear(256) + ReLU + Dropout(0.2) + Linear(4)
- 保存策略：保存验证集上准确率最高的模型权重

**输出文件：**

- `model/best.pt` — 完整的 PyTorch 模型（含结构和权重）
- `model/class_map.json` — 类别名到索引的映射

#### train_detection.py

**核心参数：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--epochs` | 50 | 训练轮数，建议 50-200 |
| `--img-size` | 640 | 输入图片尺寸，减小到 320 可加速 |
| `--batch-size` | 16 | 批大小，显存不足时减小 |
| `--model` | yolov8n | 模型大小：n(nano最快)/s/small/m/l(large最准) |
| `--patience` | 15 | 早停耐心值，连续 N 轮无改善则停止 |
| `--resume` | False | 从上次中断处继续训练 |

**训练策略：**

- 数据增强：Mosaic(4 图拼接)、MixUp(混合增强)、HSV 色调/饱和度/亮度调整、随机旋转/平移/缩放/翻转
- 优化器：自动选择 AdamW（小数据集）或 SGD（大数据集）
- 早停机制：patience=15，避免过拟合
- 每 10 个 epoch 自动保存一次中间权重

**输出文件：**

- `model/best_multi.pt` — YOLOv8 最佳权重
- `model/runs/detect/train/` — 训练过程图表（损失曲线、mAP 曲线等）
- `model/runs/detect/train/weights/last.pt` — 最新权重（用于 resume）

### 五、模型优化与微调指南

#### 提升分类准确率

1. **增加数据量** — 天池数据集有 40 个小类，映射到 4 大类后每类数千张，比增强数据效果好很多。

2. **调整学习率** — 如果训练初期 loss 下降很慢，尝试 `--lr 0.01`；如果 loss 震荡，尝试 `--lr 0.0001`。

3. **增加训练轮数** — 30 epoch 是起步，50-100 epoch 通常能获得更好的收敛效果。

4. **使用更强的模型** — 在 `train_classification.py` 的 `create_model()` 函数中，将 `mobilenet_v2` 替换为 `resnet50` 或 `efficientnet_b0`：

```python
# 替换为 ResNet50（更大但更准）
model = models.resnet50(weights=models.ResNet50_Weights.IMAGENET1K_V2)
in_features = model.fc.in_features
model.fc = nn.Sequential(nn.Dropout(0.3), nn.Linear(in_features, 4))
```

5. **类别不平衡处理** — 如果某个类别样本远多于其他类别，可以在 CrossEntropyLoss 中添加权重：

```python
class_weights = torch.tensor([1.0, 2.0, 3.0, 1.5]).to(device)  # 按类别调整
criterion = nn.CrossEntropyLoss(weight=class_weights)
```

#### 提升检测精度

1. **标注更精确的 bbox** — 这是提升检测模型效果最有效的方式。使用标注页面对图片进行多目标精确标注，替换全图 bbox 基线数据。

2. **增大模型** — 从 `yolov8n`（nano）升级到 `yolov8s`（small）或 `yolov8m`（medium），精度显著提升但推理速度变慢。

3. **调整图片尺寸** — 使用 `--img-size 640` 或 `--img-size 1280`，更大的输入尺寸有助于检测小目标。

4. **超参数调优** — YOLOv8 内置了超参数搜索功能：

```bash
# 超参数进化（自动搜索最佳超参数组合）
python -X utf8 training/train_detection.py --epochs 100 --model yolov8s
```

5. **冻结部分层** — 数据量较少时，冻结 YOLOv8 的 backbone 前几层，只训练检测头，可以防止过拟合。

6. **数据增强策略** — 在 `train_detection.py` 中调整增强参数。如果数据量小，增大 mosaic 和 mixup 强度；如果数据量大，适当减小增强避免过拟合。

#### 推理速度优化

1. **模型导出** — 将模型导出为 ONNX 或 TensorRT 格式，推理速度提升 2-5 倍：

```python
# YOLOv8 导出 ONNX
from ultralytics import YOLO
model = YOLO("model/best_multi.pt")
model.export(format="onnx", imgsz=640)
```

2. **量化** — 将模型从 FP32 转为 INT8 量化，在边缘设备上推理速度大幅提升。

3. **模型蒸馏** — 用大模型（YOLOv8m）指导小模型（YOLOv8n）训练，小模型精度更高。

### 七、常见问题

**Q: 训练时 CUDA Out of Memory 怎么办？**
减小 `--batch-size`（比如从 32 降到 16 或 8），或减小 `--img-size`（从 640 降到 320）。

**Q: 模型加载后还是 mock 模式？**
确认 `model/best.pt` 和 `model/best_multi.pt` 存在，且后端启动时没有报错。可以查看启动日志中的 "MobileNetV2 loaded" 和 "YOLOv8 loaded" 信息。

**Q: 实时识别计数不准怎么办？**
调整前端实时识别页面的「去重间隔」滑块（默认 5 秒）。同一类别在间隔内不会重复计数，切换类别会立即计数。

**Q: 标注页面的图片不显示？**
确认该记录是由检测模型识别产生的（标注页默认只显示检测模型记录），勾选「显示全部记录」可以看到所有图片。

**Q: 导出格式选哪个？**
如果后续用本项目的 YOLOv8 训练脚本，选「YOLO 检测」。如果用其他框架（Detectron2 等），选「COCO 检测」。
